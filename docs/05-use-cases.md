## 5.9 Anti-Use Cases (What Ryha AI Doesn't Do)

### ‚ùå Explicitly Out of Scope

1. **Active Exploitation:** No actual vulnerability exploitation
2. **Production Disruption:** No testing that could cause downtime
3. **Compliance Certification:** Provides evidence, not certification
4. **Legal Authorization:** Users must provide their own authorization letters
5. **Physical Security:** No physical penetration testing
6. **Social Engineering:** No human manipulation or deception
